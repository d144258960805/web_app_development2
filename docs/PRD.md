## 1. 專案概述

### 1.1 背景與動機

對大學生與日常生活中想要管理個人財務的人來說，「記帳」是養成理財習慣最基礎也最重要的一步。然而，市面上許多記帳工具功能過於複雜，或是需要付費訂閱，對於只想簡單記錄收支的使用者來說門檻過高。

本專案旨在打造一款**簡潔、直覺、免費**的網頁版記帳簿系統，讓使用者能快速記錄每日收入與支出，並透過視覺化圖表了解自己的消費習慣，進而培養良好的理財觀念。

### 1.2 目標用戶

| 用戶類型 | 描述 |
|---------|------|
| 大學生 | 每月有固定生活費或打工收入，想追蹤錢花到哪裡去了 |
| 社會新鮮人 | 剛開始自己管理財務，需要簡單工具建立記帳習慣 |
| 日常記帳者 | 任何想要記錄生活收支、掌握財務狀況的一般使用者 |

### 1.3 核心價值主張

- **簡單好用**：不需要學習複雜操作，打開即可記帳
- **一目了然**：透過餘額統計與圓餅圖，快速掌握財務全貌
- **分類管理**：自訂收支類別，讓每筆紀錄井然有序
- **零成本**：完全免費的網頁應用，無需安裝任何軟體

## 2. 功能需求

### 功能一：記錄收入與支出，顯示餘額統計

> **使用者故事**：作為一位大學生，我希望能快速新增一筆收入或支出紀錄，並即時看到目前的總餘額，以便隨時掌握自己還剩多少錢可以花。

**功能說明**：
- 使用者可新增一筆交易紀錄，需填寫：金額、類型（收入／支出）、類別、日期、備註
- 首頁顯示目前總餘額（總收入 − 總支出）
- 可查看交易紀錄列表，支援依日期排序
- 支援編輯與刪除已建立的交易紀錄

---

### 功能二：自訂收支分類

> **使用者故事**：作為一位日常記帳者，我希望能自訂收入與支出的類別（如：餐飲、交通、娛樂、薪水等），以便更精確地分類每筆交易，事後回顧時更容易分析。

**功能說明**：
- 系統預設常用類別（如：餐飲、交通、娛樂、購物、薪水、獎學金等）
- 使用者可自行新增、編輯、刪除自訂類別
- 每筆交易紀錄必須選擇一個類別
- 類別分為「收入類別」與「支出類別」兩種

---

### 功能三：圓餅圖分析月度收支

> **使用者故事**：作為一位想要改善消費習慣的使用者，我希望能透過圓餅圖看到某個月各類別的收支佔比，以便找出哪些類別花費最多，進而調整消費行為。

**功能說明**：
- 提供月度收支分析頁面
- 以圓餅圖呈現當月各類別的支出佔比
- 以圓餅圖呈現當月各類別的收入佔比
- 可切換查看不同月份的分析結果
- 圖表下方顯示各類別的金額與百分比明細

---

### 功能四：交易紀錄搜尋與篩選

> **使用者故事**：作為一位記帳一段時間的使用者，我希望能依照日期範圍、類別或關鍵字搜尋過去的交易紀錄，以便快速找到特定的消費紀錄進行檢視或修改。

**功能說明**：
- 支援依日期範圍篩選交易紀錄
- 支援依類別篩選（僅顯示某一類別的紀錄）
- 支援依收入／支出類型篩選
- 支援依備註關鍵字搜尋
- 篩選結果即時更新顯示

---

### 功能五：月度收支摘要報表

> **使用者故事**：作為一位想要定期檢視財務狀況的使用者，我希望能看到每月的收支摘要（總收入、總支出、結餘），以便追蹤自己每個月是存到錢還是超支。

**功能說明**：
- 提供月度摘要頁面，以表格或卡片形式呈現各月數據
- 每月顯示：總收入、總支出、當月結餘（收入 − 支出）
- 以顏色區分正結餘（盈餘）與負結餘（赤字）
- 可查看近 12 個月的趨勢

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 技術選擇 |
|------|---------|
| 後端框架 | Flask (Python) |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML + CSS + JavaScript |
| 圖表套件 | Chart.js（用於圓餅圖等視覺化） |

### 3.2 效能考量

- 頁面載入時間應在 2 秒內完成
- 資料庫查詢應針對常用欄位（日期、類別、類型）建立索引
- 單一使用者資料量預估在數千筆交易紀錄內，SQLite 足以應付

### 3.3 安全考量

- 所有使用者輸入需進行驗證與清理，防止 SQL Injection 與 XSS 攻擊
- 金額欄位需驗證為正數數值
- 日期欄位需驗證格式正確性

### 3.4 使用性考量

- 介面需支援繁體中文
- 操作流程應直覺，新使用者無需教學即可上手
- 頁面需具備基本的響應式設計（RWD），支援手機瀏覽
